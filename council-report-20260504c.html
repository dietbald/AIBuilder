AIBuilder Council #3 — Architecture Consistency Review

2026-05-04  ·  5 advisors  ·  5 peer reviewers  ·  Chairman synthesis
Question: Do the ARCHITECTURE.md description, the agent prompt files, and the startup scripts describe and implement a consistent system? Are there contradictions between what the architecture promises and what agents actually do? Gaps where a mechanism is described but not implemented?

The One Thing to Do First

Remove STATUS.md write instructions from dev-reviewer.md and dev-implementer.md. Every other fix operates on a system whose ground truth can be corrupted at any moment by an agent writing STATUS.md outside the Conductor's control. Until the single-writer contract is enforced — not just declared — the file-as-truth foundation the entire architecture rests on is compromised. No other fix matters if the state file cannot be trusted.

Where the Council Agrees

3 unanimous failures — every advisor converged independently

Cross-model verification is fiction. ARCHITECTURE.md promises the Reviewer runs on a different model from the Implementer. dev-conductor.md hardcodes --model claude-opus-4-7 for all sub-agents with no role-based selection. The architectural property does not exist in the shell that actually runs.
STATUS.md has multiple writers violating its own contract. ARCHITECTURE.md: "The Conductor writes it. Nothing else writes to it." dev-reviewer.md and dev-implementer.md both instruct agents to update STATUS.md directly. The Conductor uses atomic sed-and-mv. Concurrent agent writes can corrupt the file.
14-minute dead window after session rotation. Co-Conductor audits every 15 minutes. A Conductor that dies immediately after rotation goes undetected for up to 14 minutes. Architecture implies smooth recovery. Timing math does not support this.

Confirmed Implementation Bugs

Specific code-level defects found

Bug $MAX_RETRIES undefined in watchdog block. The timeout watchdog checks [ "$NEW_COUNT" -gt "$MAX_RETRIES" ] but MAX_RETRIES is only defined in the retry logic section above — not in the watchdog's scope. The comparison evaluates against an empty string. The watchdog escalation path is broken.
Bug AGENT_SCHEMA parsing: 3 fields parsed, 5 defined. dev-conductor.md "Reading Agent Output" extracts verdict, output_path, notes only. AGENT_SCHEMA.md defines 5 fields — also status and blocking_count. The Conductor cannot distinguish status: blocked from status: done. Blocked agents are treated as completed.
Gap Double-dispatch check missing = prefix. ARCHITECTURE.md explicitly specifies tmux has-session -t "=session-name" for exact-match (required to prevent partial-name false positives). dev-conductor.md hard rule #3 omits the =. Partial-name collisions are possible.
Bug review-failed is an unhandled pipeline state. dev-reviewer.md instructs the Reviewer to write review-failed to STATUS.md on FAIL. ARCHITECTURE.md's pipeline state machine has no such state. The Conductor's Action Decision Tree has no handler for it. Pipeline stalls silently — Conductor sees unrecognized state and logs idle indefinitely.

Blind Spots the Council Caught

Found only in peer review — missed by all 5 advisors

Blind spot No termination condition. The architecture documents a loop with no exit criterion for "all features done," no mechanism to stop cron, and no completion handoff. The system runs indefinitely. Manual intervention required to stop.
Blind spot RETRIES.md integrity during rotation. If session rotation triggers mid-retry-increment (between the read and write of RETRIES.md), the new Conductor inherits an incorrect count. Retry exhaustion logic becomes unreliable.
Blind spot tmux send-keys payload truncation. Shell quoting, prompt length, and tmux line-length limits can silently truncate agent instructions. The failure is non-deterministic and produces no error signal.
Blind spot tmux server restart / OS reboot wipes in-flight state. STATUS.md partial writes, the current tick counter, and active agent sessions are lost. No recovery mechanism documented for this failure path — the most likely real-world failure.
Blind spot dev-test-author.md is orphaned. Listed as Agent 4 in README. Conductor has no test-authoring status and no dispatch step. Pipeline goes implementing → reviewing directly. Either dead code or an incomplete pipeline stage.

Where the Council Clashes

The Expansionist vs. everyone else

The Expansionist called AGENT_SCHEMA "tight and fully consistent" and the cross-model firewall "a strength being undersold." The Executor's line-level measurement directly contradicted both claims: the Conductor parses 3 of 5 schema fields, and the dispatch code hardcodes the same model for all agents. The Expansionist was grading on intent; the Executor graded on implementation. Every peer reviewer sided with the Executor.

The Expansionist was right that the file-as-truth pattern is architecturally sound and underexploited. But a sound design violated in implementation is not a strength — it is a liability with good bones.
