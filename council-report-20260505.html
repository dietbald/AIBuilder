🔴 Where the Council Agrees — CONFIRMED BLOCKERS

The load-bearing unknown: claude --print --agent <file> has never been verified to work. If the flag combination is invalid, every sub-agent dispatch silently fails and the pipeline is a no-op. This must be answered empirically before anything else matters.

Four bugs confirmed by file reads (Contrarian + Reviewer 4 both read the source):

  • FATAL git add -p in dev-deployer.md — interactive TUI blocks headless --print session permanently. Watchdog kills it after 30 min every time. Pipeline can never reach staging.
  • FATAL dev-spec-verifier.md writes STATUS.md directly (lines 96–97) — violates single-writer contract. Race condition with Conductor writes guaranteed.
  • FATAL Co-conductor restart injects raw STATUS.md via tmux send-keys — newlines become Enter, pipes break quoting, shell state corrupted on any real project.
  • HIGH Env vars not exported to sub-agent tmux sessions — PROJECT_DIR, AIBUILDER_DIR, PROJECT_NAME are empty in sub-agent environments.
⚡ Where the Council Clashes

Factual dispute resolved by file read: The Executor claimed FEATURES.md format mismatch — inception outputs ### Feature Name, not ## F-01-slug, so the devloop-start.sh regex never matches. Reviewer 4 confirmed this is wrong — devloop-start.sh lines 74-80 handle both format variants. The bug doesn't exist. The Executor invented it.

Unresolved: The Contrarian claims dev-auditor.md says "Best CLI: Gemini 2.5 Pro" but the Conductor has no Gemini dispatch path. Nobody confirmed this by reading the actual dispatch logic. Lower priority than the confirmed fatals but worth checking.

Epistemic lesson: The Contrarian read files; its findings held up. The Executor listed conclusions without file reads; its unique finding was wrong. Specificity with grounding beats lists.

🔍 Blind Spots the Council Caught
  • Idempotency on restart. What happens when devloop-start.sh runs against a project with STATUS.md already mid-flight? Does the conductor resume or overwrite? Real projects get interrupted. Untested and undocumented.
  • Silent failure is operationally invisible. The watchdog detects conductor death. It does not detect sub-agent stalls. A headless pipeline that is silently failing looks identical to one running normally. No alerting mechanism exists for "agent X has been silent for N minutes."
  • The system has never been run end-to-end. Not a code review problem — an empirical unknown. No one has watched a feature go from FEATURES.md to deployed code. Every analysis so far has been theoretical. "Ready" has no acceptance criterion.
📋 The Recommendation

No. It is not ready to run on a real project.

The blockers are execution failures, not documentation gaps. An interactive command that permanently hangs headless sessions (git add -p), a multi-writer race on the state file (spec-verifier), and a tmux injection that corrupts shell state (co-conductor restart) will each independently stop a real run. Fix any one and the others remain.

The Expansionist's vision — franchised AI factory, publishable architecture — is downstream of a system that can complete one feature end-to-end. That test has not been run. Run it first.

⚡ The One Thing to Do First

Verify the CLI invocation works. Open a terminal and run:

claude --print --agent /path/to/AIBuilder/.claude/agents/dev-spec-author.md "test"

If it errors: The entire dispatch architecture needs to be redesigned. Find what flag combination works for headless agent invocation and rebuild everything around that.

If it succeeds: Confirmed. Then fix in this order: (1) replace git add -p with git add -A in dev-deployer.md, (2) remove STATUS.md write from dev-spec-verifier.md, (3) escape the co-conductor STATUS.md tmux injection, (4) export env vars in the sub-agent dispatch block. Then run one feature, live, on a throwaway repo.

Everything else is conjecture until that command returns a result.

First Principles  Verify CLI before all else

CLI invocation claude --print --agent <file> is unverified. If it doesn't work, the pipeline is a no-op dressed in scaffolding.

Environment not inherited across tmux sessions. API keys, PATH, working dirs don't propagate automatically.

State machine has no stuck-feature timeout. Silent sub-agent failure leaves feature in its current stage forever. Watchdog catches conductor death, not this.
